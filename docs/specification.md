# 帯分割補正ドキュメントスキャンアプリ 仕様書

## 1. 概要

本アプリは、スマートフォンで撮影した本・ノート・書類のページ画像を補正するAndroidアプリである。

従来のドキュメントスキャンアプリでは、撮影画像上の四隅を指定し、その四角形全体を1枚の矩形へ射影変換する方式が一般的である。しかし、本のページのように奥行き方向へ湾曲している紙面では、ページ全体を1枚の平面として扱うと、文字や罫線の一部に歪みが残りやすい。

本アプリでは、ページを横方向に複数の帯へ分割し、各帯ごとに個別に補正することで、四隅補正だけでは直しきれない湾曲を軽減する。

本アプリの補正方式を、以下では **帯分割補正** と呼ぶ。

---

## 2. 基本コンセプト

### 2.1 従来方式

従来の四隅補正では、元画像上のページ領域を1つの四角形として指定し、それを補正後画像上の1つの長方形へ変換する。

```text
元画像上の四角形
↓
補正後画像上の長方形
```

この方式は、紙面がほぼ平面である場合には有効である。

しかし、本のページのように紙面が湾曲している場合、ページ全体を1枚の平面として扱うことに無理がある。

---

### 2.2 本アプリの方式

本アプリでは、ページを横方向に分割し、複数の縦長の帯として扱う。

```text
元画像上のページ
↓
横方向に複数の補正帯へ分割
↓
各補正帯ごとに補正後画像上の対応領域へ変換
↓
1枚の補正後画像として結合
```

これにより、ページ全体を1枚の平面として補正するのではなく、局所的に異なる補正を行える。

---

## 3. 画面構成

画面構成は以下の4画面を基本とする。

```text
1. 写真撮影画面
   ↓
2. ページ基準点指定画面
   ↓
3. 元画像編集画面
   ↓↑
4. 補正結果画面
```

写真撮影後、すぐに元画像編集画面へ進むのではなく、先にページ基準点を指定する。

---

## 4. 写真撮影画面

### 4.1 役割

補正対象となる画像を取得する画面である。

### 4.2 機能

- 写真を撮影する
- 端末内の画像を選択する

MVP段階でも、画像選択機能は入れておくと開発・デバッグがしやすい。

---

## 5. ページ基準点指定画面

### 5.1 役割

元画像上で、補正対象ページの基準となる4点を指定する画面である。

### 5.2 指定する点

ページ基準点として、以下の4点を指定する。

- 左上
- 右上
- 右下
- 左下

この4点から、以下を決定する。

- 左端境界
- 右端境界
- 補正対象ページの初期範囲
- 補正後画像のデフォルト縦横比
- 元画像編集画面で使う初期座標系

### 5.3 画面遷移

```text
写真撮影または画像選択
↓
ページ基準点指定
↓
元画像編集画面
```

ページ基準点指定画面では、通常のスキャンアプリに近い操作でページ四隅を合わせる。  
元画像編集画面では、この4点をもとに、左端境界・右端境界・補正帯枠を表示する。

---

## 6. 元画像編集画面

### 6.1 役割

元画像上で、補正対象となる入力領域を編集する画面である。

この画面では、主に以下を行う。

- 分割ラインの追加・削除
- 分割ラインの位置調整
- 端点接続線による補正帯枠の確認
- 幅配分バーによる出力境界の調整
- 補正結果画面への遷移

### 6.2 表示要素

元画像編集画面には、以下を表示する。

- 元画像
- 左端境界
- 右端境界
- 分割ライン
- ライン端点
- 端点接続線
- 補正帯枠
- 幅配分バー
- 境界マーカー
- 補正結果を見るボタン

ページ外枠は、四角形の外枠として常時強調表示するのではなく、左端境界・右端境界、および端点接続線によって補正帯の形が分かるように表示する。

画面イメージ：

```text
┌────────────────────┐
│ 元画像               │
│                    │
│   |------/------|   │
│   |     /       |   │
│   |    /        |   │
│   |---/---------|   │
│                    │
├────────────────────┤
│ |------o------o---| │
├────────────────────┤
│ [補正結果を見る]     │
└────────────────────┘
```

---

## 7. 補正結果画面

### 7.1 役割

現在の入力境界と出力境界に基づいて生成された補正後画像を表示する画面である。

この画面は基本的には確認用だが、補正後画像を見ながら以下を調整できる。

- 幅配分バーによる各補正帯の出力幅
- 出力外枠による補正後画像全体の縦横比

また、補正後画像上には、各補正帯が写る矩形領域の境界として **出力境界線** を表示できる。出力境界線はオン・オフ可能とする。

### 7.2 表示要素

補正結果画面には、以下を表示する。

- 補正後画像
- 出力外枠
- 幅配分バー
- 境界マーカー
- 出力境界線
- 出力境界線の表示オン・オフ切り替え
- 元画像編集画面に戻るボタン
- 保存ボタン

画面イメージ：

```text
┌────────────────────┐
│ 補正後画像           │
│ ┌────────────────┐ │
│ │ |      |     |  │ │
│ │ |      |     |  │ │
│ │ |      |     |  │ │
│ └────────────────┘ │
├────────────────────┤
│ |------o------o---| │
├────────────────────┤
│ [境界表示] [分割を調整] [保存] │
└────────────────────┘
```

## 8. 用語定義

### 8.1 元画像

撮影または選択された、補正前の画像。

---

### 8.2 補正後画像

現在の入力境界・出力境界・出力外枠に基づいて生成された画像。

---

### 8.3 ページ基準点

元画像上で、補正対象ページの基準となる4点。

ページ基準点は以下の4点で定義される。

- 左上
- 右上
- 右下
- 左下

この4点によって、ページの左端境界・右端境界、および初期状態の補正対象範囲を定義する。

従来の意味ではページ外枠に相当するが、本アプリの元画像編集画面では、ページ外枠を四角形として常時強調表示するのではなく、左端境界・右端境界と端点接続線によって補正帯枠を表示する。

---

### 8.4 左端境界・右端境界

ページ基準点から定義される固定の入力境界。

左端境界は、ページ基準点の左上点と左下点を結ぶ線である。  
右端境界は、ページ基準点の右上点と右下点を結ぶ線である。

左端境界はページ基準点の左上点と左下点、右端境界は右上点と右下点を結ぶ入力境界である。
各端点を個別に移動できるほか、境界線本体をドラッグすることで、2つの端点を同じ移動量だけ動かして境界線全体を平行移動できる。

---

### 8.5 幅配分バー

補正後画像における横方向の幅配分を編集するためのバー。

幅配分バー上には境界マーカーが配置される。

例：

```text
|--------o------o----|
```

この例では、境界マーカーが2つあるため、補正後画像は3つの補正帯に分割される。

---

### 8.6 境界マーカー

幅配分バー上に表示される丸い操作点。

境界マーカーは、**補正後画像上の出力境界の位置**を表す。

重要な点として、境界マーカーは入力境界の位置を表すものではない。ただし、各境界マーカーは、元画像上の対応する分割ラインと紐づいている。

```text
境界マーカー
= 出力境界の位置を表すUI

分割ライン
= 入力境界の位置を表すUI
```

---

### 8.7 分割ライン

元画像上に表示される縦方向の線。

分割ラインは、元画像上の入力境界の位置を表す。

1本の分割ラインは、1つの境界マーカーと対応する。

---

### 8.8 ライン端点

分割ラインの上端・下端にある操作点。

ライン端点を個別にドラッグすることで、分割ラインの角度を調整できる。

```text
上端点
  |
  |
下端点
```

---

### 8.9 端点接続線

隣り合う入力境界の上端点同士、または下端点同士を結ぶ線。

端点接続線は、元画像上で補正帯の上辺・下辺を表す。

たとえば、隣り合う2本の入力境界がある場合、上端点同士を結ぶ線がその補正帯の上辺になり、下端点同士を結ぶ線がその補正帯の下辺になる。

---

### 8.10 補正帯枠

隣り合う2本の入力境界と、上下の端点接続線によって囲まれる四角形。

補正帯枠は、元画像上で1つの補正帯として変換される領域を視覚的に示す。

```text
左入力境界       右入力境界
    |--------------|  ← 上側の端点接続線
    |              |
    |              |
    |--------------|  ← 下側の端点接続線
```

---

### 8.11 入力境界

元画像側における補正帯の境界。

入力境界には以下が含まれる。

- 左端境界
- 分割ライン
- 右端境界

分割ラインは、ユーザーが操作可能な入力境界である。  
左端境界・右端境界は、ページ基準点から定義される固定の入力境界である。

---

### 8.12 出力境界

補正後画像側における補正帯の境界。

出力境界には以下が含まれる。

- 補正後画像の左端
- 境界マーカーが表す位置
- 補正後画像の右端

境界マーカーは、ユーザーが操作可能な出力境界である。

---

### 8.13 出力境界線

補正後画像上に表示される縦方向の線。

出力境界線は、補正後画像上における補正帯同士の境界を表す。

各出力境界線は、幅配分バー上の境界マーカーと対応する。境界マーカーを移動すると、対応する出力境界線の位置も変化する。

出力境界線は、補正結果画面で表示オン・オフを切り替えられる。

```text
補正結果画面上：
出力境界線 = 補正後画像上の補正帯の境界

幅配分バー上：
境界マーカー = 出力境界線の位置を操作する点
```

---

### 8.14 出力外枠

補正結果画面上で、補正後画像全体の矩形範囲を表す枠。

出力外枠は、補正後画像の縦横比を調整するためのUIでもある。

ユーザーは補正結果画面で出力外枠の辺をドラッグし、補正後画像全体の縦横比を調整できる。

出力外枠は編集補助用のオーバーレイであり、保存される補正後画像には含めない。

---

### 8.15 補正帯

隣り合う入力境界に挟まれた元画像上の領域。

より正確には、隣り合う2本の入力境界と、それらの上端点同士・下端点同士を結ぶ端点接続線によって定まる四角形領域を補正帯と呼ぶ。

補正処理では、各補正帯を補正後画像上の対応する横幅領域へ変換する。

たとえば、境界マーカーが2つある場合、補正帯は3つになる。

```text
出力側：
|--------o------o----|
  帯1       帯2   帯3
```

---

### 8.16 エラー領域

補正帯の射影変換が数学的または数値的に計算できない場合に、補正結果画面上へ表示する代替領域。

エラー領域は、目立つ色や市松模様などで描画し、その補正帯が正常に補正できなかったことを示す。

ユーザーが手動でエラー領域を作成する機能は設けない。  
エラー領域は、補正処理時に数学的に変換不能と判断された場合のみ発生する。

---

### 8.17 帯分割補正

元画像を複数の補正帯に分割し、各補正帯ごとに補正後画像上の対応領域へ変換する補正方式。

---

## 9. 境界マーカー・分割ライン・出力境界線の関係

1つの境界には、以下の2種類の位置情報がある。

```text
出力側の位置：
境界マーカーの位置

入力側の位置：
分割ラインの位置
```

境界マーカーは出力境界を表す。  
分割ラインは入力境界を表す。

両者は同じ境界IDで紐づいている。

```text
境界ID 1
├ 出力境界：幅配分バー上の境界マーカー
└ 入力境界：元画像上の分割ライン
```

補正結果画面で出力境界線を表示する場合、出力境界線は境界マーカーと同じ出力境界を、補正後画像上に縦線として表示したものである。

```text
境界ID 1
├ 出力境界UI：幅配分バー上の境界マーカー
├ 出力境界表示：補正後画像上の出力境界線
└ 入力境界表示：元画像上の分割ライン
```

---

## 10. 補正後画像のサイズ・縦横比

### 10.1 デフォルトの縦横比

補正後画像の縦横比は、デフォルトではページ基準点から推定する。

推定には、ページ基準点から得られる上辺・下辺・左辺・右辺の長さを用いる。

```text
topWidth     = distance(topLeft, topRight)
bottomWidth  = distance(bottomLeft, bottomRight)
leftHeight   = distance(topLeft, bottomLeft)
rightHeight  = distance(topRight, bottomRight)

estimatedWidth  = (topWidth + bottomWidth) / 2
estimatedHeight = (leftHeight + rightHeight) / 2

aspectRatio = estimatedWidth / estimatedHeight
```

この推定値を、補正後画像の初期縦横比として使う。

---

### 10.2 補正結果画面での縦横比調整

補正結果画面では、出力外枠の辺をドラッグすることで、補正後画像全体の縦横比を調整できる。

操作例：

```text
右辺を外側へドラッグ：
補正後画像を横長にする

右辺を内側へドラッグ：
補正後画像を縦長寄りにする

下辺を下へドラッグ：
補正後画像を縦長にする

下辺を上へドラッグ：
補正後画像を横長寄りにする
```

MVPでは、最低限、右辺または下辺のドラッグによって縦横比を調整できればよい。

---

### 10.3 出力外枠と境界マーカーの関係

境界マーカーの `outputX` は、0.0〜1.0 の正規化座標として保持する。

実際の出力位置は、補正後画像の横幅に応じて決まる。

```text
actualOutputX = outputX * outputWidth
```

そのため、出力外枠を変更して補正後画像の横幅が変化しても、境界マーカーの相対位置は維持される。

---

### 10.4 出力サイズ

保存時には、現在の出力外枠の縦横比に基づいて、最終的な補正後画像を生成する。

MVPでは、出力解像度は以下のいずれかの方針とする。

```text
案1：
プレビュー用の基準横幅を決め、縦横比から高さを計算する。

案2：
元画像の解像度をもとに、補正後画像の横幅・高さを推定する。

案3：
固定の横幅を使い、縦横比から高さを計算する。
```

MVPでは、まず固定の横幅を使い、縦横比から高さを計算する方針でよい。

---

## 11. 幅配分バーの操作仕様

### 11.1 初期状態

初期状態では、境界マーカーは存在しない。

```text
|--------------------|
```

この状態では、補正帯は1つであり、通常の四隅補正に近い処理になる。

---

### 11.2 境界マーカーの追加

幅配分バー上の空いている場所を長押しすると、長押し位置の近くにコンテキストメニューを表示する。

空き領域を長押しした場合のメニュー項目：

- ここに境界を追加

この項目を選択すると、長押し位置に対応する `outputX` を持つ境界マーカーを追加する。

例：

```text
|--------------------|
          ↓
|--------o-----------|
```

境界マーカーを追加すると、それに対応する分割ラインが元画像上に追加される。

境界マーカーの追加時には、既存の境界マーカーおよび左右端との最小間隔制約を満たす位置のみ追加可能とする。

---

### 11.3 境界マーカー追加時の分割ライン初期位置

新しく追加された分割ラインは、新しい境界マーカーの `outputX` と、その左右に隣接する入力境界・出力境界の対応関係から初期位置を決定する。

新しい境界の `outputX` を `x` とし、出力側で直近の左境界位置を `xL`、直近の右境界位置を `xR` とする。

補間比率 `t` は次のように求める。

```text
t = (x - xL) / (xR - xL)
```

左側の入力境界の上端点・下端点を `leftTop`、`leftBottom`、右側の入力境界の上端点・下端点を `rightTop`、`rightBottom` とすると、新しい分割ラインの初期位置は次のように決定する。

```text
inputTop    = leftTop    + t * (rightTop    - leftTop)
inputBottom = leftBottom + t * (rightBottom - leftBottom)
```

つまり、新しい分割ラインは、出力側での相対位置と同じ比率で、現在の隣接入力境界間を内分する位置に配置する。

たとえば、すでに次の境界が存在するとする。

```text
outputX = 0.2
outputX = 0.6
```

この間の `outputX = 0.3` に新しい境界を追加する場合、

```text
t = (0.3 - 0.2) / (0.6 - 0.2)
  = 0.25
```

となる。

したがって、新しい分割ラインは、`outputX = 0.2` の分割ラインから `outputX = 0.6` の分割ラインへ25%進んだ位置に配置する。

```text
0.2       0.3                 0.6
 |---------|-------------------|
      25%
```

上端点と下端点は、それぞれ独立に同じ比率 `t` で線形補間する。

左側に内部境界が存在しない場合は、左端境界を `outputX = 0.0` の境界として使用する。

右側に内部境界が存在しない場合は、右端境界を `outputX = 1.0` の境界として使用する。

そのため、内部境界が1本も存在しない初期状態では、左端境界と右端境界の間を補間して最初の分割ラインを生成する。

この方式により、既存の分割ラインや左右端境界が手動調整されている場合でも、新しい分割ラインは現在の隣接境界形状に沿った位置へ追加される。

---

### 11.4 境界マーカーの移動

境界マーカーは、幅配分バー上を左右にドラッグできる。

境界マーカーを移動すると、対応する出力境界の位置が変わる。  
つまり、補正後画像における各補正帯の幅配分が変わる。

境界マーカーの移動は、対応する分割ラインの位置を直接変更しない。

補正結果画面で出力境界線の表示がオンの場合、境界マーカーの移動に合わせて、対応する出力境界線の位置も変化する。

---

### 11.5 境界マーカーの削除

境界マーカーを長押しすると、長押し位置の近くにコンテキストメニューを表示する。

境界マーカーを長押しした場合のメニュー項目：

- 削除

この項目を選択すると、境界マーカーを削除する。

境界マーカーを削除すると、同じ境界IDを持つ対応する分割ラインも削除される。

削除後は、削除された境界の左右に存在していた入力境界が新たに隣接境界となり、その間が1つの補正帯として扱われる。

残存する境界の `outputX`、`inputTop`、`inputBottom` は変更しない。

長押ししただけで即削除はせず、コンテキストメニューの「削除」を選択した場合のみ削除する。

---

### 11.6 境界マーカーの順序制約

境界マーカーは、他の境界マーカーをまたいで移動できない。

たとえば、以下の状態で、

```text
|---o1-----o2---|
```

`o1` を `o2` の右側へ移動することはできない。

また、補正帯の幅が極端に狭くならないように、境界マーカー同士の最小間隔を設定する。

例：

```text
minGap = 0.03 ～ 0.05
```

---

## 12. 元画像編集画面の操作仕様

### 12.1 分割ラインおよび左右端境界の表示

境界マーカーを追加すると、元画像上に対応する分割ラインを表示する。

境界マーカーが2つある場合、分割ラインも2本表示される。

---

### 12.2 補正帯枠の表示

元画像編集画面では、分割ラインだけでなく、各入力境界の端点同士を結んだ線も表示する。

具体的には、隣り合う入力境界について、以下を表示する。

- 上端点同士を結ぶ端点接続線
- 下端点同士を結ぶ端点接続線

これにより、各補正帯が元画像上でどの四角形領域として補正されるのかを視覚的に確認できる。

ページ基準点による外枠は、四角形として常時強調表示するのではなく、左端境界・右端境界と端点接続線によって補正帯枠として表示する。

---

### 12.3 分割ラインおよび左右端境界全体のドラッグ

分割ラインおよい左右端境界全体をドラッグすると、ラインを平行移動する。

内部的には、上端点と下端点を同じ移動量だけ移動する。

```text
inputTop    += delta
inputBottom += delta
```

ラインを移動すると、それに接続している端点接続線も更新される。

---

### 12.4 ライン端点の個別ドラッグ

分割ラインおよび左右端境界の上端点または下端点を個別にドラッグできる。

これにより、ラインの角度を調整できる。

```text
上端点だけ移動
↓
分割ラインの角度が変わる
↓
接続する端点接続線も更新される
```

---

### 12.5 ズーム・パン操作

元画像編集画面では、元画像をズーム・パンできるようにする。

操作の優先順位は以下とする。

1. ライン端点を触った場合は、ライン端点の移動操作とする
2. 分割ライン本体を触った場合は、分割ライン全体の平行移動操作とする
3. 何もない場所をドラッグした場合は、画像のパン操作とする
4. 2本指ピンチ操作は、画像のズーム操作とする

分割ラインやライン端点の座標は、表示座標ではなく元画像の座標系で保持する。

```text
inputTop / inputBottom：
元画像のピクセル座標で保持

表示時：
現在のズーム・パン状態に合わせて画面座標へ変換
```

---

### 12.6 分割ラインの交差

元画像上で分割ライン同士が交差しても、MVPでは特に制限しない。

理由は以下の通り。

- 原理上、補正後画像を生成できる場合がある
- 分割ラインと補正帯枠は元画像上に重ねて表示されているため、変な位置関係になっていればユーザーが視覚的に判断できる
- 交差判定や移動制限を入れると、実装と操作感が複雑になる

したがって、MVPではライン交差に対する警告や制限は設けない。

ただし、数学的に射影変換を計算できない補正帯が発生した場合は、補正結果生成時にエラー領域として描画する。

---

## 13. 補正結果画面の操作仕様

### 13.1 補正後画像の表示

補正結果画面では、現在の入力境界・出力境界・出力外枠に基づいて生成した補正後画像を表示する。

---

### 13.2 幅配分バーの表示

補正結果画面にも、元画像編集画面と同じ幅配分バーを表示する。

補正結果画面で境界マーカーを移動すると、補正後画像における各補正帯の幅配分が変わる。

この操作により、補正後画像を見ながら各補正帯の幅を調整できる。

---

### 13.3 出力境界線の表示オン・オフ

補正結果画面では、補正後画像上に出力境界線を重ねて表示できる。

出力境界線は、各補正帯が補正後画像上で割り当てられている矩形領域の境界を表す縦線である。

出力境界線は、幅配分バー上の境界マーカーと対応する。境界マーカーを移動すると、対応する出力境界線の位置も移動する。

出力境界線の表示はオン・オフ可能とする。

表示オンの場合：

- 補正後画像上に各出力境界線を表示する
- どの補正帯の幅を調整しているか分かりやすくする
- 幅配分バーによる調整結果を視覚的に確認しやすくする

表示オフの場合：

- 補正後画像だけを確認できる
- 境界線なしの最終的な見た目を確認できる

出力境界線は編集補助用のオーバーレイであり、保存される補正後画像には含めない。

---

### 13.4 出力外枠の操作

補正結果画面では、補正後画像全体の矩形範囲を示す出力外枠を表示する。

ユーザーは出力外枠の辺をドラッグすることで、補正後画像の縦横比を調整できる。

MVPでは、最低限以下のいずれかを実装する。

```text
右辺をドラッグ：
横幅を調整する

下辺をドラッグ：
高さを調整する
```

出力外枠の変更に応じて、補正後画像を再生成する。

出力外枠は編集補助用のオーバーレイであり、保存される補正後画像には含めない。

---

### 13.5 エラー領域の表示

補正帯の射影変換が数学的または数値的に計算できない場合、その補正帯に対応する出力領域をエラー領域として描画する。

エラー領域は、目立つ色や市松模様など、通常の補正結果と明確に区別できる見た目にする。

ユーザー操作によって手動でエラー領域を作る機能は設けない。  
エラー領域は、補正処理時に変換不能と判定された場合のみ表示する。

---

### 13.6 元画像編集画面への戻り

補正結果を確認したうえで、分割ラインや補正帯枠を再調整したい場合は、元画像編集画面へ戻れるようにする。

---

### 13.7 保存

保存ボタンを押すと、現在の入力境界・出力境界・出力外枠に基づいて最終的な補正後画像を生成し、保存する。

以下の編集補助用オーバーレイは保存画像には含めない。

- 出力外枠
- 出力境界線
- 幅配分バー
- 境界マーカー

---

## 14. 補正処理の考え方

### 14.1 境界列

補正処理では、以下を入力境界・出力境界の列として扱う。

入力境界：

```text
左端境界
分割ライン1
分割ライン2
...
右端境界
```

出力境界：

```text
補正後画像左端
境界マーカー1
境界マーカー2
...
補正後画像右端
```

境界マーカーと分割ラインは、同じ境界IDで対応する。

---

### 14.2 補正帯の生成

隣り合う入力境界同士に挟まれた領域を、1つの補正帯とする。

より正確には、隣り合う2本の入力境界と、それぞれの上端点同士・下端点同士を結ぶ端点接続線によって定まる四角形領域を、1つの補正帯とする。

たとえば、境界マーカーが2つある場合は、以下の3つの補正帯ができる。

```text
補正帯1：左端境界 ～ 分割ライン1
補正帯2：分割ライン1 ～ 分割ライン2
補正帯3：分割ライン2 ～ 右端境界
```

---

### 14.3 各補正帯の変換

各補正帯について、元画像側の四角形領域を、補正後画像側の長方形領域へ変換する。

たとえば、ある補正帯について、入力側の四角形は以下で定義される。

```text
leftInputTop
rightInputTop
rightInputBottom
leftInputBottom
```

出力側の長方形は以下で定義される。

```text
leftOutputX, 0
rightOutputX, 0
rightOutputX, outputHeight
leftOutputX, outputHeight
```

各補正帯に対して、個別に射影変換を行う。

---

### 14.4 変換不能ケース

ユーザー操作としては、分割ラインの交差や不自然な補正帯を制限しない。

ただし、補正結果生成時に、各補正帯について射影変換が計算可能かを確認する。

以下のような場合、射影変換が失敗する、または数値的に不安定になる可能性がある。

- 隣り合う入力境界が完全に重なる
- 補正帯の4点が一直線またはほぼ一直線になる
- 補正帯の面積がゼロまたは極端に小さい
- 射影変換行列を計算できない

射影変換を計算できる場合は、結果が大きく歪んでもそのまま生成する。

射影変換を計算できない場合は、その補正帯に対応する出力領域をエラー領域として描画する。

---

### 14.5 補正後画像の生成タイミング

補正結果画面へ遷移するときに、現在の入力境界・出力境界・出力外枠に基づいて補正後画像を生成する。

補正結果画面上で境界マーカーを動かした場合も、補正後画像を再生成する。

補正結果画面上で出力外枠を変更した場合も、補正後画像を再生成する。

ただし、処理負荷を抑えるため、以下の方針を取る。

```text
編集中：
低解像度プレビューを生成

保存時：
元解像度または高解像度で最終補正画像を生成
```

ドラッグ中に毎回再生成すると重い場合は、指を離したタイミングで再生成する。

---

## 15. データ構造案

### 15.1 境界ペア

境界マーカーと分割ラインは、同じ境界IDで紐づく。

コード上では、入力境界と出力境界の対応関係を表すデータとして、`BoundaryPair` を持つと分かりやすい。

```kotlin
data class BoundaryPair(
    val id: Long,
    var outputX: Float,
    var inputTop: PointF,
    var inputBottom: PointF
)
```

各プロパティの意味は以下。

```text
id:
境界を識別するID

outputX:
補正後画像上の出力境界の横位置
幅配分バー上の境界マーカーが表す値
0.0 ～ 1.0 の正規化座標

inputTop:
元画像上の入力境界の上端点
分割ラインの上端点

inputBottom:
元画像上の入力境界の下端点
分割ラインの下端点
```

---

### 15.2 固定境界

左端境界・右端境界は、固定境界として扱う。

左端境界：

```text
outputX = 0.0
inputTop = pageTopLeft
inputBottom = pageBottomLeft
```

右端境界：

```text
outputX = 1.0
inputTop = pageTopRight
inputBottom = pageBottomRight
```

補正処理時には、固定境界と `BoundaryPair` の一覧をまとめて、`outputX` の昇順に並べる。

---

### 15.3 補正帯

補正帯は、隣り合う2つの境界から生成される。

```kotlin
data class CorrectionBand(
    val leftBoundary: BoundaryLine,
    val rightBoundary: BoundaryLine
)
```

固定境界と分割ラインを同じ形式で扱うため、補正処理時には以下のような境界線データに変換すると扱いやすい。

```kotlin
data class BoundaryLine(
    val outputX: Float,
    val inputTop: PointF,
    val inputBottom: PointF
)
```

---

### 15.4 出力設定

補正後画像の縦横比や出力サイズは、出力設定として保持する。

```kotlin
data class OutputSettings(
    var aspectRatio: Float,
    var outputWidth: Int,
    var outputHeight: Int
)
```

MVPでは、`outputWidth` を固定値として持ち、`aspectRatio` から `outputHeight` を計算してもよい。

```text
outputHeight = outputWidth / aspectRatio
```

---

### 15.5 表示設定

補正結果画面の表示補助設定を保持する。

```kotlin
data class ResultViewSettings(
    var showOutputBoundaryLines: Boolean
)
```

## 22. 現時点の最終方針

本アプリでは、フルメッシュ編集やパラメータ式補正ではなく、以下の方針を採用する。

```text
- 横方向の帯分割に限定する
- 写真撮影後にページ基準点を指定する
- ページ基準点から補正後画像の初期縦横比を推定する
- 境界マーカーで出力境界を調整する
- 分割ラインで入力境界を調整する
- 境界マーカーと分割ラインを1対1で紐づける
- 境界追加時は、出力側の相対位置に基づいて隣接入力境界間を線形補間し、分割ラインの初期位置を決める
- 元画像編集画面では補正帯枠を表示する
- 補正結果画面では出力境界線をオン・オフ表示できる
- 補正結果画面では出力外枠をドラッグして縦横比を調整できる
- 補正後画像は帯ごとの射影変換で生成する
- 変換不能な補正帯はエラー領域として描画する
- 補正結果画面でも幅配分バーを操作できるようにする
```

この方式により、ユーザーは以下の操作を直感的に行える。

```text
ページ基準点を動かす：
補正対象ページの基準範囲を決める

分割ラインを動かす：
元画像のどこを切り出すかを調整する

境界マーカーを動かす：
補正後画像で各補正帯をどれくらいの幅にするかを調整する

出力外枠を動かす：
補正後画像全体の縦横比を調整する
```

また、元画像編集画面では補正帯枠を表示することで、どの四角形領域が1つの補正単位として扱われるのかを確認できる。

補正結果画面では出力境界線を表示することで、境界マーカーによる幅配分調整が補正後画像上のどの領域に対応しているかを確認できる。

これにより、四隅補正より柔軟で、フルメッシュ編集より操作負担の少ない補正アプリを目指す。
